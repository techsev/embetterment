What's this site about?

Glad you asked. My name is Seve Teruel, I’m an up and coming Web developer working just outside of Toronto and living even further outside of Toronto. I've spent the last year working for a publishing company that mainly deals with selling financial and health publications through email campaigns. I've started this site as a way to show case my progress as both a web developer as well as a human being.

I’ve been listening to lots of different web development podcasts as of late, so that I can know what I don’t know, and learn it. Stuff like CSS preprocessors, NODE.js, Responsive images. e.t.c. I’ve been dying to try out these new tech trends on projects at work, however, there usually isn't enough time to implement them. So instead I decided to build this website as a sandbox to try out ideas as well as a showcase of the skills I do have.

What I wanted to do was have a way to quickly and dynamicly have progress bars tracking my skills and goals that could easily be modified by changing 1 - 2 variables. I considered making it database driven or using PHP however, my goal for this site is to try techniques i’ve never used before, including making an entire static website.

So instead I decided to use SASS and Compass. That way I can make changes fairly quickly on, compile the CSS and then push to git pages.

Where the CSS differs for each progress bar is the width, color and the percentage amount.

I created a SASS mixin that when given a percentage, would set it as the inner-div’s width, the “content” field of the div:after class as well as modify the background color of the inner div between the colors red and green.

Then all I have to do was supply the required percentage for each bar, which in some cases I just supplied directly or in others, I used SASS variables and math to generate the percentage. I had a little fun with my SASS Progress Bar Variables
